## Migration Blocker
PostgREST (the Supabase REST API) does NOT expose PostgreSQL system catalog tables
(pg_tables, pg_policies, pg_policies) in its schema cache. This means the audit
script cannot query these tables directly via the service role client.

Solution implemented:
1. Created migration: supabase/migrations/20260330000001_rls_audit_helper.sql
   - Creates public.rls_audit_tables  (mirrors pg_tables for public schema)
   - Creates public.rls_audit_policies (mirrors pg_policies for public schema)
   - Populates both tables from pg_tables/pg_policies at migration time
2. Updated scripts/test-rls-policies.ts to query rls_audit_tables and rls_audit_policies
3. Script gracefully detects missing migration and shows manual apply instructions

## Key Finding
Supabase PostgREST schema cache does NOT include PostgreSQL system catalogs
(pg_tables, pg_policies, information_schema, pg_* functions). Only user-created
tables in public schema are exposed. The workaround (audit helper tables) is
the correct approach per Supabase docs.
